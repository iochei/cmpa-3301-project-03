**Prepared By:** Bradley Ochei

## 1.0 What went well?
Reflecting on the semester's work, particularly the transition from the midterm project to this final portfolio, the adoption of a disciplined documentation process was highly successful. By formally defining the Project Scope Statement (Project 03) before writing a single line of code, I had a clear roadmap. This prevented the confusion and aimless experimentation that characterized my initial approach to the midterm. Furthermore, committing to version control via GitHub Desktop early and often (as listed in the Task Schedule) ensured that my progress was saved remotely, mitigating the risk posed by my laptop's hardware limitations. This structured approach saved significant time in the final implementation phase.

## 2.0 What went wrong?
The most significant challenge during the execution of the portfolio project arose from overestimating my capacity to include complex content. Initially, I intended to build a multi-page site with individual, deeply detailed case studies for five projects. As I began coding the HTML, I realized that writing five distinct, high-quality case studies was far more time-consuming than anticipated. This threatened to delay my adherence to the schedule constraint. I had to scale back the depth of content for the final submission to ensure the site launched on time, demonstrating a failure in realistic estimation during the planning phase.

## 3.0 What would you do differently?
If I were to undertake this project again, I would implement a stricter "Content First" workflow. Before touching HTML/CSS, I would fully complete the drafting and editing of all portfolio project descriptions and my "About Me" biography in a separate document. In this project, I found myself trying to write copy and code simultaneously, which fragmented my focus. By finalizing all content assets first, the technical implementation phase would become purely about structure and styling, rather than composition. This would have allowed me to meet my original goal of 5 deep case studies without risking the deadline.
