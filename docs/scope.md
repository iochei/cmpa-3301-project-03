# Project Scope Statement: Bradley Ochei - Portfolio
**Project Name:** Final Project 03: Personal Portfolio Website
**Date:** July 23, 2026 (Updated)
**Prepared By:** Bradley Ochei
**Course:** CMPA 3301

---

## 1.0 Project Justification (Business Case)
The student requires a centralized, professional online presence to showcase their academic and personal project work. This project is justified by the need to demonstrate technical proficiency in front-end web development to potential employers and to provide a professional platform for networking and career opportunities post-graduation.

## 2.0 Project Product Scope Description (High-Level Characteristics)
The project will result in a responsive, single-page or multi-page portfolio website utilizing HTML5, CSS3, and vanilla JavaScript. The site will feature:
1.  **Homepage/Landing:** A professional introduction, brief mission statement, and quick navigation.
2.  **About Me:** A detailed biography, skills section (technical/design tools), and a link to download the resume.
3.  **Portfolio Projects:** A gallery displaying 3-5 detailed project case studies, including descriptions, technologies used, and links to live demos or GitHub repositories.
4.  **Contact:** A functional contact method (email link or direct contact form).
5.  **Deployment:** Via GitHub Pages.

## 3.0 Project Deliverables (The Tangible Output)
1.  **Documentation:** This approved Project Scope Statement (scope.md).
2.  **Project Plan:** Including risk analysis and task schedule (plan.md).
3.  **Retrospective:** Reflection on the process (retrospective.md).
4.  **Source Code:** All HTML, CSS, and project images/media.
5.  **Deployed Website:** The live site accessible at a public URL.
6.  **Final Presentation:** A brief presentation (slide deck) of the finished product.

## 4.0 Acceptance Criteria
The project is successfully completed when:
*   All content listed in Section 3.0 is delivered and operational.
*   The website passes basic accessibility checks (e.g., proper color contrast and image alt text).
*   The site is fully responsive on mobile devices (tablet and smartphone breakpoints).
*   All links to external project repositories and live demos are functional.
*   All source code is committed and pushed to the designated GitHub repository.
*   The project is submitted via the Canvas LMS on or before the due date of July 24th.

## 5.0 Project Exclusions (What is OUT of scope?)
*   A backend database (no user data storage or login system).
*   A blog or CMS (all content is static).
*   Integration with complex APIs (e.g., fetching live Instagram feeds).
*   Password protection for any part of the site.
*   Professional SEO services (beyond basic meta tags).

## 6.0 Project Constraints
*   **Schedule:** Final deadline for deployment and submission is July 24th.
*   **Technology:** Restricted to HTML, CSS, and vanilla JavaScript (no frameworks like React/Vue).
*   **Resources:** Individual project, $0 budget.
*   **Hardware:** Development is constrained by the student's current laptop hardware (low disk space).

## 7.0 Project Assumptions
*   It is assumed that the student has the necessary project assets (previous project screenshots, code samples, professional summary text) prepared prior to coding.
*   It is assumed that the student will dedicate approximately 5-8 hours per week to meet the schedule constraint.
*   It is assumed that the technical workaround (GitHub Desktop) will remain operational for submissions to bypass local installer issues.
